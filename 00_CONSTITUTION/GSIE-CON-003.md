# GSIE-CON-003 — La Connaissance avant le Code

Édition : Première Édition
Version : 1.0 (Validated)
Statut : Validé par le Fondateur
Classification : Loi Fondamentale (Immuable)

## Principe

La connaissance constitue le véritable cœur de GSIE. Le code n'est qu'un
moyen de structurer, conserver, enrichir et transmettre cette
connaissance.

Toute architecture, tout moteur, toute interface et tout algorithme
existent au service de la connaissance. Aucune décision technique ne
peut compromettre la qualité scientifique du patrimoine de connaissances.

## Pourquoi

Les technologies évoluent rapidement — langages, frameworks, modèles
d'IA — tandis que les connaissances forestières constituent un
patrimoine durable. GSIE est conçu pour survivre aux changements
technologiques. Si le code est le patrimoine principal, un changement
de technologie détruit le patrimoine. Si la connaissance est le
patrimoine principal, le code peut être réécrit sans perte.

## Conséquences

- Les modèles conceptuels précèdent le code.
- Les ontologies précèdent les bases de données.
- Les relations scientifiques précèdent les algorithmes.
- Les sources sont documentées avant toute implémentation.
- Le code n'est pas le patrimoine principal — il est remplaçable.

## Patrimoine GSIE

Le patrimoine comprend :

- connaissances scientifiques ;
- graphes de connaissances ;
- ontologies ;
- référentiels ;
- modèles conceptuels ;
- règles de corrélation ;
- documentation fondatrice.

Le code n'est pas dans cette liste. Le code sert le patrimoine, il n'est
pas le patrimoine.

## Exemple

Avant le Diagnostic Engine doivent exister : modèle stationnel, modèle
pédologique, modèle climatique, modèle des essences, règles de
corrélation, niveaux de preuve. Coder le Diagnostic Engine sans ces
modèles produirait un moteur qui devine au lieu de raisonner.

## Contre-exemple

Un développeur code un moteur de recommandation d'essences directement
à partir de données empiriques non documentées, sans modèle conceptuel
préalable. Le moteur fonctionne sur les cas testés mais ne peut pas
être audité ni étendu. C'est une violation de CON-003. Le code doit être
précédé d'un modèle conceptuel documenté.
