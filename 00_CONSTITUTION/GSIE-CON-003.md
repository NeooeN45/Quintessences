# GSIE-CON-003 — La Connaissance avant le Code

Édition : Première Édition
Version : 1.0 (Draft)
Statut : À valider
Classification : Loi Fondamentale (Immuable)

## Déclaration
La connaissance constitue le véritable cœur de GSIE. Le code n'est qu'un moyen de structurer, conserver, enrichir et transmettre cette connaissance.

## Principe fondamental
Toute architecture, tout moteur, toute interface et tout algorithme existent au service de la connaissance. Aucune décision technique ne peut compromettre la qualité scientifique du patrimoine de connaissances.

## Pourquoi cette loi existe
Les technologies évoluent rapidement tandis que les connaissances forestières constituent un patrimoine durable. GSIE est conçu pour survivre aux changements de langages, frameworks et modèles d'IA.

## Conséquences
- Les modèles conceptuels précèdent le code.
- Les ontologies précèdent les bases de données.
- Les relations scientifiques précèdent les algorithmes.
- Les sources sont documentées avant toute implémentation.

## Patrimoine GSIE
Le patrimoine comprend :
- connaissances scientifiques ;
- graphes de connaissances ;
- ontologies ;
- référentiels ;
- modèles conceptuels ;
- règles de corrélation ;
- documentation fondatrice.

Le code n'est pas le patrimoine principal.

## Cas d'application
Avant le Diagnostic Engine doivent exister :
- modèle stationnel ;
- modèle pédologique ;
- modèle climatique ;
- modèle des essences ;
- règles de corrélation ;
- niveaux de preuve.

## Anti-Lois
Interdiction :
- de coder une connaissance non documentée ;
- de créer un moteur sans modèle conceptuel ;
- de cacher une connaissance dans le code ;
- de contourner la documentation.

## Déclaration finale
« Le code est temporaire. La connaissance constitue le patrimoine durable de GSIE. »
