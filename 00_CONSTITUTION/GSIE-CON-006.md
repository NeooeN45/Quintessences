# GSIE-CON-006 — La Documentation fait partie du Produit

Édition : Première Édition
Version : 1.0 (Draft)
Classification : Loi Fondamentale (Immuable)

## Déclaration
La documentation n'est pas un livrable secondaire : elle constitue une partie intégrante du produit GSIE.

## Principe fondamental
Aucun composant ne peut être considéré comme terminé sans une documentation permettant de comprendre son objectif, son fonctionnement, ses dépendances et ses limites.

## Pourquoi cette loi existe
Le code évolue. La documentation transmet la connaissance et garantit la maintenabilité.

## Obligations
- Toute fonctionnalité possède une spécification.
- Tout moteur possède une documentation technique.
- Toute donnée possède une définition.
- Tout algorithme possède une justification scientifique.
- Toute API possède une documentation d'utilisation.

## Anti-Lois
- Développer sans documentation.
- Laisser une documentation obsolète.
- Documenter uniquement le code.

## Déclaration finale
« Un composant non documenté est un composant inachevé. »
