# GSIE Engineering Handbook
# Tome I — Vision & Fondation

## Préface

GSIE (GeoSylva Intelligence Engine) est né d'un constat simple : les outils forestiers actuels excellent souvent dans une tâche précise (cartographie, inventaire, SIG, calculs), mais très peu sont capables de relier l'ensemble des connaissances nécessaires à une décision forestière.

L'objectif de GSIE n'est pas de remplacer le forestier. Son objectif est de lui fournir un système expert transparent, explicable et scientifiquement fondé.

Ce manuel constitue la référence officielle du projet. Toute évolution de GSIE devra rester compatible avec les principes décrits ici.

---

# Chapitre 1 — Pourquoi GSIE existe

## 1. Le problème

La connaissance forestière est dispersée entre des centaines de livres, guides techniques, publications scientifiques, bases de données, cartes et retours d'expérience.

Un gestionnaire doit aujourd'hui croiser manuellement :
- les données de terrain ;
- la topographie ;
- les sols ;
- la réserve utile maximale (RUM) ;
- le climat actuel ;
- les projections climatiques ;
- les contraintes réglementaires ;
- les habitats naturels ;
- les connaissances bibliographiques.

Aucun outil ne réalise aujourd'hui cette synthèse de manière totalement explicable.

## 2. La réponse de GSIE

GSIE a pour ambition de devenir une plateforme de connaissance forestière.

Ses missions sont :

- centraliser les connaissances ;
- qualifier leur niveau de preuve ;
- les relier dans un graphe de connaissances ;
- raisonner sur ces connaissances ;
- produire des recommandations justifiées ;
- conserver la décision finale à l'utilisateur.

## 3. Les principes

1. La science avant l'opinion.
2. La connaissance avant le code.
3. L'explicabilité avant la performance.
4. La documentation avant l'implémentation.
5. La modularité avant la complexité.

## 4. Les objectifs à long terme

À terme, GSIE devra être capable de :
- réaliser un diagnostic stationnel semi-automatique ;
- proposer des essences adaptées au climat futur ;
- expliquer chaque recommandation ;
- intégrer les données IGN, INPN, Météo-France, Copernicus et les référentiels scientifiques ;
- fonctionner hors ligne sur le terrain ;
- servir de plateforme commune aux applications GeoSylva.

## Conclusion

GSIE n'est pas conçu comme une application.

GSIE est conçu comme une infrastructure scientifique durable dont GeoSylva constitue la première implémentation.
