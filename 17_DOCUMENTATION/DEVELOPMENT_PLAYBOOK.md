# GSIE Development Playbook

## Mission
Définir la méthode officielle de développement de GSIE.

## Cycle
Vision → RFC → Architecture → Spécification → Implémentation → Tests → Documentation → Validation.

## Règles
- La Constitution prévaut toujours.
- Développement incrémental.
- Tests systématiques.
- Documentation minimale mais à jour.
- Aucun code sans objectif métier identifié.

## Definition of Done
Code testé, documenté, validé, traçable et conforme à la Constitution.
