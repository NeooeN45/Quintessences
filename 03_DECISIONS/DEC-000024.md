# DEC-000024 — Ingestion des données forestières ONF/CNPF/IGN

| Champ | Valeur |
|---|---|
| **ID** | DEC-000024 |
| **Statut** | Proposé |
| **Date** | 2026-07-16 |
| **Décideur** | Camille Perraudeau (Fondateur) |
| **RFC liée** | RFC-0013 (ingestion données ONF/CNPF/IGN) |
| **ADR liés** | ADR-001 (racine resource), ADR-002 (Temporal Engine) |
| **Datasets liés** | DS-001 (BD Forêt v2), DS-003 (IFN), DS-005 (RPF), DS-011 (BDAT), DS-012 (RPFR) |

## Décision

Valider la RFC-0013 et autoriser l'implémentation du module
`gsie_api.ingestion` pour l'ingestion structurée des datasets
forestiers français (ONF, CNPF, IGN, INRAE) dans l'API GSIE.

## Contenu

1. **Module `gsie_api.ingestion`** — pipeline d'ingestion générique
2. **Datasets P0 (Vague 2)** : BD Forêt v2 (DS-001), IFN (DS-003),
   catalogues stations CNPF
3. **Datasets P1 (Vague 3)** : RPF/RPFR (DS-005/DS-012), BDAT (DS-011),
   LiDAR HD (DS-002)
4. **Endpoints API ingestion** — `/api/v1/ingestion/datasets/{ds_id}/run`
5. **Traçabilité CON-010** — Revision v1 + Provenance pour chaque record
6. **Mapping métamodèle v6.2** — chaque dataset mappe vers des types
   spécifiques (place, observation, concept, assertion, result, entity)

## Motivation

Les moteurs GSIE (Forest Dynamics, Botanical, Pedology, GIS) nécessitent
des données forestières réelles françaises pour fonctionner. Sans
ingestion de BD Forêt v2 et de l'IFN, les simulations ne peuvent pas
être calibrées. Les catalogues stations CNPF sont essentiels pour les
recommandations sylvicoles.

## Impact

- **Code** : nouveau module `gsie_api/ingestion/` (6 fichiers P0)
- **Stockage** : +20 Go PostgreSQL (estimation P0)
- **Tests** : couverture 80% sur le module ingestion
- **API** : 4 nouveaux endpoints ingestion
- **Mémoire** : mise à jour PROJECT_MEMORY.md, ROADMAP.md, CHANGELOG.md

## Alternatives rejetées

1. **Ingestion manuelle (scripts ad hoc)** — non reproductible, non
   tracée (violation CON-010)
2. **ETL externe (Airflow/Prefect)** — complexité opérationnelle
   excessive pour Vague 2
3. **Ingestion directe SQL (COPY/shp2pgsql)** — contourne l'API, pas
   de traçabilité Revision

## Risques

- Volume BD Forêt v2 (50 Go) → ingestion par batch (10 000 records)
- Qualité géométrique → validation PostGIS `ST_IsValid`
- Changements de schéma source → tests de régression
