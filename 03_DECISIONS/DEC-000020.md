## Contexte

La Semaine 3 de la Vague 1 (Fondations) implémente le Knowledge Engine,
deuxième moteur dans la chaîne d'intelligence : Evidence → Knowledge.

Le Knowledge Engine est la **source unique de vérité** pour tous les
moteurs de raisonnement (KNOWLEDGE_ENGINE.md §6). Il centralise, structure
et versionne les connaissances scientifiques qualifiées.

## Décision

Implémenter le Knowledge Engine en **Python pur** (stockage en mémoire
pour la Vague 1), avec les fonctionnalités suivantes :

1. **Ingestion** — reçoit les connaissances au statut « accepte » depuis
   l'Evidence Engine, rejette quarantine et refuse (CON-001).
2. **Requête** — 5 types : par_concept, par_relation, par_domaine,
   par_essence, par_station, avec filtres et pagination.
3. **Versionnement** (CON-010) — chaque révision archive l'ancienne version
   dans l'historique avec justification. Aucune connaissance supprimée
   silencieusement.
4. **Révision** — mise à jour du contenu, niveau de preuve, source ou
   domaines de validité, avec justification obligatoire.

## Justification du choix Python

Contrairement à l'Evidence Engine (cœur Rust pour la performance de
l'évaluation), le Knowledge Engine est un moteur de **stockage + requête**
(pas de calcul scientifique intensif). Python pur est suffisant pour la
Vague 1. La persistance PostgreSQL/Neo4j et les pistes de recherche
(RDF2Vec, GraphRAG, PROV-O — KNOWLEDGE_ENGINE.md §8) sont des évolutions
futures.

## Endpoints API

| Méthode | Endpoint | Description |
|---|---|---|
| GET | `/api/v1/knowledge/status` | Statut du moteur |
| GET | `/api/v1/knowledge/version` | Version et backend |
| POST | `/api/v1/knowledge/ingest` | Ingère une connaissance (201) |
| POST | `/api/v1/knowledge/query` | Interroge le graphe (200) |
| POST | `/api/v1/knowledge/revise` | Révise une connaissance (200/404/400) |
| GET | `/api/v1/knowledge/stats` | Statistiques du graphe |

## Qualité

- 33 nouveaux tests (19 unitaires + 14 API)
- 155 tests au total (122 existants + 33 nouveaux)
- Ruff + mypy --strict : zéro erreur
- Rate limiting : 30/min (ingest), 60/min (query)
- Auth JWT obligatoire sur tous les endpoints sensibles

## Contrat d'interface respecté

Conforme à KNOWLEDGE_ENGINE.md §5 :
- Entrée : `KnowledgeIngestRequest` (depuis Evidence Engine, statut « accepte »)
- Sortie : `KnowledgeQueryResult` (liste de `KnowledgeObject`)
- Requête : `KnowledgeQuery` (type + filtres + evidence_min + pagination)
