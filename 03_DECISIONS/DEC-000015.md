# DEC-000015 — Unification du système d'articles constitutionnels (adoption RFC-0002, Option A)

| Champ | Valeur |
|---|---|
| **ID** | DEC-000015 |
| **Date** | 2026-07-13 |
| **Statut** | ACTIVE |
| **Décideur** | Camille Perraudeau (Fondateur) |
| **RFC d'origine** | RFC-0002 (Proposé 2026-07-06 → Adopté 2026-07-13) |
| **Lois fondatrices** | GSIE-CON-000 (primauté), GSIE-CON-005 (traçabilité), GSIE-CON-010 (historique) |

---

## Décision

Le Fondateur **adopte RFC-0002, Option A** : les fichiers
`GSIE-CON-0XX.md` deviennent l'**unique source de vérité** du corpus
d'articles constitutionnels.

Le gabarit de 100 fichiers vides `ARTICLE_001.md` à `ARTICLE_100.md`
(0 octet chacun) est **supprimé**. Un index de référence
(`ARTICLES_INDEX.md`) le remplace, listant les articles `GSIE-CON-0XX`
sans dédoubler les fichiers.

## Contexte

- Deux systèmes de nommage coexistaient pour un même corpus : 100
  fichiers `ARTICLE_0XX.md` vides et 11 fichiers `GSIE-CON-0XX.md`
  rédigés (000 Locked, 001-010 Validated).
- Le livrable 010 du ROADMAP pointait vers les fichiers vides, rendant
  le corpus réel invisible pour un lecteur suivant la structure
  officielle — incohérence de traçabilité (CON-005).
- Vérification préalable : les 100 fichiers `ARTICLE_0XX.md` étaient
  tous vides (0 octet). Leur suppression ne perd aucun contenu.

## Impact

| Document / Dossier | Changement |
|---|---|
| `00_CONSTITUTION/ARTICLE_001.md` → `ARTICLE_100.md` | Supprimés (100 fichiers vides) |
| `00_CONSTITUTION/ARTICLES_INDEX.md` | Créé — index de référence des articles |
| `00_CONSTITUTION/README.md` | Section « Ce qui peut y être ajouté » corrigée |
| `00_CONSTITUTION/GSIE-CON-000.md` (Locked) | Non touché |
| `02_RFC/RFC-0002.md` | Statut → Adopté |
| `ROADMAP.md` | Livrable 010 : mention du gabarit `ARTICLE_0xx` retirée |
| `PROJECT_MEMORY.md` | RFC-0002 marqué Adopté |
| `CHANGELOG.md` | Entrée ajoutée |

## Garde-fous

- `GSIE-CON-000.md` (Locked) n'a pas été modifié.
- La numérotation `GSIE-CON-0XX` n'est pas plafonnée à 100 : de
  nouveaux articles peuvent être ajoutés par RFC dédié.
- L'index `ARTICLES_INDEX.md` ne porte aucun contenu normatif — il
  renvoie aux fichiers source.
