# DEC-000005 — Amendement DEC-000003 : archivage du code du banc Ignis

## Décision

Le Fondateur **amende** DEC-000003 (point 5) et DEC-000004 (section « Banc
de simulation Ignis ») pour autoriser l'**archivage du code du banc de
simulation Ignis** dans le dépôt GSIE, en tant que **preuve de concept du
Jalon 0**, sous `22_PROJECT_MEMORY/Ignis/`.

## Contexte

DEC-000003 et DEC-000004 disposaient que le banc de simulation vit
**hors du dépôt GSIE** (`~/Ignis/` WSL2) et ne produit **pas de code dans
le dépôt**. Cette règle visait à protéger la pureté documentaire de la
Phase 1.

La Phase 1 étant **clôturée** (DEC-000004) et le projet entré en Phase 2
(Architecture), le Fondateur juge utile de **conserver une trace versionnée**
du code du banc dans le dépôt GSIE pour :

1. **Reproductibilité** — capter l'état initial du banc (Jalon 0) comme
   référence évolutive.
2. **Traçabilité** — l'historique Git du dépôt GSIE devient le journal
   complet du projet, banc inclus.
3. **Continuité** — éviter la perte de contexte entre sessions et machines.

## Contenu acté

1. **Périmètre archivé** : les fichiers exécutables du Jalon 0 présents dans
   `22_PROJECT_MEMORY/Ignis/` sont **intégrés au dépôt GSIE** :
   - `premier_vol.py` — script de premier vol drone (MAVSDK).
   - `plot_front.py` — visualisation du front de feu.
   - `plot_demo.sh`, `run_aullene.sh`, `run_demo.sh`, `setup_bashrc.sh` —
     scripts d'orchestration du banc.

2. **Statut documentaire** : ces fichiers sont des **artefacts d'archive du
   Jalon 0**, pas du code métier des 14 moteurs GSIE. Ils ne déclenchent pas
   l'implémentation des moteurs (réservée Phase 4).

3. **Garde-fous maintenus** :
   - Aucun de ces fichiers n'est un moteur GSIE.
   - Le banc opérationnel continue de vivre dans `~/Ignis/` (WSL2) ; le
     dépôt n'en est qu'une **archive ponctuelle**.
   - L'interdiction de code métier GSIE dans le dépôt (Phase 4) reste
     entière.

4. **Amendement explicite** :
   - DEC-000003 point 5 : la phrase « Le banc de simulation (`~/Ignis/`
     WSL2) est séparé du dépôt GSIE et ne produit pas de code dans le dépôt »
     est **amendée** : le banc reste séparé opérationnellement, mais une
     **archive versionnée** du Jalon 0 est désormais conservée dans le dépôt.
   - DEC-000004 « Banc vit hors du dépôt » : idem — le banc vit hors dépôt,
     son **archive Jalon 0** est dans le dépôt.

## Conséquences

- `22_PROJECT_MEMORY/Ignis/` contient désormais à la fois les livrables
  documentaires (comparatif, guide, contexte, journal) et les artefacts
  exécutables du Jalon 0.
- La mémoire du projet (`PROJECT_MEMORY.md`, `CHANGELOG.md`) est
  synchronisée pour refléter cet amendement.
- Aucun effet sur l'interdiction de code métier GSIE (Phase 4).
