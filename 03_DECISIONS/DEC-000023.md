# DEC-000023 — Migration API GSIE vers le métamodèle v6.2

| Champ | Valeur |
|---|---|
| **ID** | DEC-000023 |
| **Statut** | Proposé |
| **Date** | 2026-07-16 |
| **Auteur** | Camille Perraudeau (Fondateur) |
| **RFC liée** | RFC-0012 (migration API v6.2) |
| **ADR liés** | ADR-001 (racine resource), ADR-002 (Temporal Engine), ADR-007 (architecture API v6.2) |
| **Supersède** | DEC-000019 (partie persistance et endpoints API) |

## Décision

Migration complète de l'API GSIE du schéma v6.1 (12 tables,
`KnowledgeObject`) vers le métamodèle v6.2 (73 types noyau, table racine
`resource`, GSIE Temporal & Provenance Engine).

## Contenu

1. **Table racine `resource`** (ADR-001) — class-table inheritance
2. **73 modèles SQLAlchemy** groupés par domaine (12 fichiers)
3. **45 enums PostgreSQL** (§3.3 à §3.22 + enums supplémentaires)
4. **Migration Alembic 0002** — création des 73 tables + migration des données existantes
5. **CRUD générique** — 7 endpoints pour les 71 types resources (ADR-007)
6. **WebSocket** — canal temps réel pour le Hub (Centre de Commandement UE5.8)
7. **Registry pattern** — `@register_type` pour auto-enregistrement des types

## Impact

- **Breaking change** : `KnowledgeObject` → `Assertion` (type 9)
- **12 tables anciennes** migrées vers les nouvelles tables v6.2
- **Endpoints `/knowledge/*`** migrés vers le CRUD générique `/resources`
- **Endpoint `/evidence/evaluate`** conservé (pas de breaking change)
- **Auth JWT RS256** conservée
- **Middlewares** (TraceId, CORS, rate limiting) conservés

## Conséquences

- L'API est alignée sur le métamodèle v6.2 (73 types)
- Le Hub (UE5.8) peut se connecter en WebSocket
- Les apps (GeoSylva, QGISIA) nécessitent un SDK (à générer)
- La Vague 2 peut implémenter les 14 moteurs sur le nouveau schéma
