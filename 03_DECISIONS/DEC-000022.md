## Décision

Le Fondateur propose d'adopter le **métamodèle v6.2 de l'Encyclopédie de
l'Écosystème** (livrable 213) tel que défini dans
`GSIE/ARCHITECTURE/ECOSYSTEM_METAMODEL.md` et justifié par la RFC-0011.
La v6.2 enrichit la v6.1 (42 types) avec 18 types issus de la passe
écologique du Fondateur (ScaleContext, Phenomenon, EcologicalProcess,
RelationType, SamplingEvent, TraitDefinition, TraitValue, Feature,
FeatureSet, Inference, Question, Hypothesis, Decision, Recommendation,
Scenario, Correlation, EcosystemService, Capability), plus 1 type pour
le GSIE Temporal & Provenance Engine (ResourceDiff, type 61), plus 4
types pour l'audit FAIR/RGPD/SOSA (Sample type 62, Consent type 63,
DataSubject type 64, PersistentIdentifier type 65), plus 8 types pour
la passe dynamiques écologiques (Flow type 66, ConfidenceGraph type 67,
Goal type 68, Constraint type 69, KnowledgeLineage type 70, Experiment
type 71, TerrainSession type 72, EcologicalState type 73), portant le
noyau à **73 types**.

Cette décision, en un bloc unique :

1. **Supersède** les livrables Validated 302, 304, 309 et 310
2. **Amende** la directive Active GSIE-DIR-0008 (§2.1, §2.3, §2.4)
3. **Amende** les décisions DEC-000012, DEC-000019 et DEC-000020
4. **Adopte** les 6 ADR-001 à ADR-006
5. **Lance** la Vague 0 (gouvernance + RFC + ADR + audit migration)

## Contexte

- La proposition v5 (110 classes) a été archivée comme non adoptée après
  audit critique (sur-ingénierie, mélange de niveaux).
- La convergence v6.1 (42 types, 5 niveaux) a été auditée : 19
  corrections identifiées (5 P0, 8 P1, 6 P2), toutes tranchées par le
  Fondateur.
- Trois contradictions P0 avec des livrables Validated (302, 309, 310)
  et une contradiction P0 avec une directive Active (DIR-0008) ont été
  identifiées. Cette DEC les résout par superseding et amendement
  explicites.
- Le numéro DEC-000022 a été libéré par l'archivage de la proposition v5
  non adoptée (voir `22_PROJECT_MEMORY/SUPERSEDED_DRAFTS/`).

## Arbitrages Fondateur intégrés

### Structurels
- **A1** : Superseding par RFC-0011 + DEC-000022 unique (un seul vote)
- **A2** : Noyau 73 types accepté (42 v6.1 + 18 v6.2 écologique + 1 Temporal Engine + 4 FAIR/RGPD/SOSA + 8 dynamiques écologiques), chacun justifié par cycle de vie + usage multidomaine
- **A3** : Racine `resource` unique (class-table inheritance, FK fortes)
- **A4** : Subagents sources (10 domaines + 10 types) — hors périmètre DEC

### Sémantiques
- **S1** : `claim_kind` 7 valeurs (observation, relation, rule, threshold, model, classification, absence)
- **S2** : Absence = `Result.value_type='absence'` + `sampling_effort`/`detection_probability` sur Observation
- **S3** : `ConflictCluster` (type 42) pour les conflits Assertion↔Assertion
- **S4** : `FeatureOfInterest` = sous-type d'Entity (`type='feature_of_interest'`)

### Techniques
- **T1** : Bitemporalité via **GSIE Temporal & Provenance Engine** (Revision + Snapshot + ResourceDiff + PROV-O, moteur métier intégré — pas d'extension externe)
- **T2** : Benchmark AGE dès Vague 1 (décision basée sur données réelles)
- **T3** : Vague 1 complète — 73 types implémentés dès le départ
- **T4** : Adaptateur Evidence : Rust évalue (A-F) + Python enrichit (évaluateur, méthode, date, scope)

### Production
- **P1** : RFC-0011 principal + annexes séparées
- **P2** : Vague 0 = RFC + 6 ADR + tests contractuels + contrats interface (~2 semaines)
- **P4** : `Source.source_nature` 6 valeurs (data_provider, knowledge_provider, reference, expert_statement, regulatory, model_output)

## Garde-fous

- La Constitution Scientifique (S-1 à S-7) prime sur toute
  considération d'architecture.
- Aucune implémentation de code ne démarre avant le gate documentaire
  de la Vague 0 (CON-003).
- Les 67 tests Rust de l'Evidence Engine sont préservés (arbitrage T4).
- Le contenu historique des documents supersédés est conservé (CON-010).
- Neo4j, Elasticsearch, Jena et GraphQL sont **différés, pas exclus** —
  réintroduits si le benchmark ou le besoin le justifie.

## Note de validation rétroactive (2026-07-16)

Le Fondateur valide cette décision rétroactivement. L'implémentation de
l'API GSIE (métamodèle v6.2, 69 modèles SQLAlchemy, migration 0002) a
démarré avant la validation formelle de cette DEC, ce qui contredit le
gate documentaire imposé ligne 143 (« aucune implémentation ne démarre
avant le gate documentaire de la Vague 0 »). Cet écart est assumé
explicitement : le travail sous-jacent a été audité (sécurité, qualité,
cohérence métamodèle) et jugé globalement solide. À l'avenir, le
gate documentaire sera respecté strictement — un garde-fou CI sera mis
en place pour bloquer les commits non testés.
