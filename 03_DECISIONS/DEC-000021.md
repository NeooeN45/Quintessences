## Contexte

La Semaine 4 clôt la Vague 1 (Fondations) en intégrant l'Evidence Engine
et le Knowledge Engine en un pipeline complet. C'est la **tranche
verticale prioritaire** du ROADMAP : la première démonstration intégrée
du système.

## Décision

Créer un module d'intégration `pipeline.py` qui chaîne les deux moteurs :

1. **Soumission** → Evidence Engine (qualification A-F)
2. **Si statut « accepte »** → Knowledge Engine (ingestion dans le graphe)
3. **Requête** → KnowledgeQueryResult (paginé, filtré)
4. **Révision** → versionnement CON-010 (historique immuable)

Les connaissances en quarantaine ou refusées sont retournées à l'appelant
pour validation humaine (CON-001) — elles ne sont pas ingérées
automatiquement.

## Validation

- 11 tests d'intégration E2E (8 engine + 3 API)
- 166 tests au total (155 + 11 nouveaux)
- Ruff + mypy --strict : zéro erreur
- Pipeline complet testé via les endpoints API :
  - evaluate → ingest → query
  - evaluate → ingest → revise → query v2
  - refuse d'ingestion d'une connaissance refusée
