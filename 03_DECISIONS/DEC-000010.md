## Décision

Le Fondateur **acte l'adoption d'Unreal Engine 5.8 + Cesium for Unreal**
comme **moteur 3D du jumeau numérique vivant** (DIR-0005). Cette décision
réalise concrètement l'ADR-001 du livrable 208 (moteur 3D interchangeable
sans logique métier) et ouvre les livrables 211 et 212.

## Justification

1. **GSIE-CON-002 (science)** : trois publications académiques récentes
   (FIRETWIN 2025, FIRE-VLM 2026, IVSR 2026) valident l'approche « jumeau
   numérique incendie dans Unreal Engine » — ce n'est pas une lubie
   isolée, c'est un domaine de recherche actif financé par la NASA et NSF.

2. **GSIE-CON-007 (modularité)** : UE 5.8 est un **client de visualisation**,
   jamais une dépendance de calcul. Le jumeau numérique tourne côté serveur
   (DIR-0006) ; UE 5.8 reflète l'état, il ne le calcule pas. ADR-001 du
   livrable 208 (moteur 3D interchangeable) reste respecté.

3. **GSIE-CON-005 (traçabilité)** : Cesium for Unreal (racheté par Bentley
   Systems en 2024) est un plugin open source adossé à un acteur
   infrastructure sérieux, pas un projet fragile.

4. **Cohérence stack** : UE 5.8 expose nativement WebSockets, HTTP et JSON
   (utilisés par Epic pour sa Remote Control API) — aucune dépendance
   externe exotique nécessaire pour l'ingestion temps réel.

## Architecture partagée Ignis ↔ GeoSylva

| Partagé (plugin commun) | Séparé (logique propre) |
|---|---|
| Cesium (terrain géoréférencé) | Niagara feu/fumée (Ignis) |
| WebSockets + JSON natif | PCG végétation (GeoSylva) |
| Conventions de données (Lambert 93, WGS 84) | Mode d'usage (temps réel vs planification) |
| Synchronisation réel/simulé | |

Recommandation : **un seul projet Unreal avec architecture en plugins
internes** (CON-007), plutôt que deux projets séparés. À formaliser en RFC
quand GeoSylva-Unreal sera activement priorisé.

## Discipline d'exécution

- **Ignis (livrable 211)** : priorité 1 — prototype WebSocket en cours,
  MVP visé.
- **GeoSylva-Unreal (livrable 212)** : **en attente volontaire** — ne pas
  construire avant qu'Ignis ait son MVP fonctionnel (règle S-08).

## Garde-fous

- RFC-0004 §8 (autonomie limitée à la navigation, reprise manuelle) reste
  applicable — UE 5.8 est un outil de visualisation, pas de commandement.
- GSIE-CON-001 (décideur humain) — le COS reste le décideur, UE 5.8 montre
  mais ne commande pas.
