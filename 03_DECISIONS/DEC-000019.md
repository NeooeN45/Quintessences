## Résumé

Validation de l'architecture technique Phase 4 après analyse par 3
subagents de recherche (sources web 2025-2026). L'approche
multi-langage (Python + Rust + Go différé) et l'architecture API
(FastAPI + PostGIS + Redis) sont **validées**. Le plan d'implémentation
initial (8 semaines) est **refondu** en plan réaliste de 24 semaines
respectant les dépendances entre moteurs.

---

## Acte 1 — Validation de l'architecture multi-langage

L'approche **Python 3.12 (FastAPI) + Rust (pyo3) + Go (différé)** est
validée.

**Justification :**
- Python + Rust couvre 95% des besoins (écosystème scientifique +
  sécurité mémoire CON-002)
- PyO3 est mature en production (Polars, Pydantic v2, Ruff — gains 7.4x)
- Go est différé car MAVSDK-Go est "Proof of concept" en 2026
  (MAVSDK-Python est "Used in production")
- Si besoin temps réel confirmé : envisager Rust axum/tokio avant Go

**Ajustements :**
- `py.allow_threads()` systématique dans bindings Rust (GIL management)
- Profiler avant de déplacer code vers Rust (FFI overhead réel sur petits inputs)

## Acte 2 — Validation de l'architecture API

L'architecture **FastAPI + asyncpg + PostGIS + Redis + OpenTelemetry**
est validée avec 5 ajustements.

**Ajustements P0 (bloquants) :**
1. Bypass PgBouncer pour connexion LISTEN/NOTIFY dédiée
2. asyncpg `statement_cache_size=0` pour transaction pooling
3. pygeoapi comme lib Starlette dans FastAPI (pas from scratch, pas standalone)

**Ajustements P1 :**
4. Architecture par modules moteurs (pas DDD pur)
5. fastgeoapi à étudier pour sécurité enterprise (OIDC + OPA)

## Garde-fous

1. **Respect strict du graphe de dépendances** (livrable 204) — aucun
   moteur ne démarre avant que ses dépendances soient testées
2. **2 semaines par moteur Rust** (réaliste pour développeur solo
   apprenant Rust)
3. **Plan de repli Python** : si un moteur Rust bloque > 3 semaines,
   implémenter en Python natif et migrer plus tard
4. **Tests d'intégration** après chaque vague (pipeline complet)
5. **Profiler avant Rust** : ne déplacer que les hot paths réels
6. **CON-001** : l'IA assiste, ne décide jamais
7. **CON-005** : traçabilité totale (OpenTelemetry + structlog)
