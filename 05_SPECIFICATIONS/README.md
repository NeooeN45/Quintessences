# 05 — Specifications

## Objectif

Formaliser les exigences fonctionnelles et non fonctionnelles de GSIE.
Une spécification décrit *ce que le système doit faire*, jamais
*comment*.

## Responsabilités

- Définir les exigences par domaine
- Établir les critères d'acceptation
- Assurer la traçabilité exigence → moteur

## Ce qui peut y être ajouté

- Spécifications fonctionnelles par domaine
- Spécifications non fonctionnelles (performance, hors-ligne, sécurité)
- Matrices de traçabilité

## Ce qui est interdit

- Décrire l'implémentation (rôle de `04_ARCHITECTURE` et `09_ENGINES`)
- Ajouter du code

## Liens

- **04_ARCHITECTURE** : répond aux spécifications
- **09_ENGINES** : implémente les spécifications
- **03_DECISIONS** : les spécifications découlent des décisions
