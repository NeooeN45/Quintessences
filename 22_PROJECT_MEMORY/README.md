# 22 — Project Memory

## Objectif

Conserver la mémoire complète du projet : décisions, idées, visions,
modifications, évolutions. Aucune décision importante ne doit être
perdue.

## Responsabilités

- Traçabilité de toutes les décisions et évolutions
- Permettre à une nouvelle IA de comprendre rapidement l'état du projet
- Conserver l'historique des visions et des journaux

## Ce qui peut y être ajouté

- `PROJECT_MEMORY.md` — vue courante
- `VISION_HISTORY.md` — évolution des visions
- `DECISION_HISTORY.md` — chronologie des décisions
- `IDEA_BACKLOG.md` — idées non encore décidées
- `FOUNDER_JOURNAL.md` — journal du fondateur
- `ARCHITECT_JOURNAL.md` — journal de l'architecte
- `Ignis.md` — registre vivant des idées Ignis (branche fonctionnelle, RFC-0004)
- `Ignis/` — sous-dossier des livrables du Jalon 0 Ignis
- Context Snapshots (toutes les 10 Directives)
- Master Context Snapshots (toutes les 100 Directives)

## Ce qui est interdit

- Supprimer une entrée d'historique (on archive, on ne supprime pas)
- Modifier rétroactivement une décision enregistrée
