# 02 — RFC (Request for Comments)

## Objectif

Formaliser les propositions d'évolution technique, scientifique ou
organisationnelle avant adoption. Un RFC est un document de débat
structuré.

## Responsabilités

- Documenter le *pourquoi* d'une évolution avant son implémentation
- Permettre le débat et la traçabilité des décisions
- Servir de référence pour les décisions (`03_DECISIONS`)

## Ce qui peut y être ajouté

- `RFC-XXXX.md` — nouvelles propositions
- Mises à jour de RFC existantes (avec historique)

## Ce qui est interdit

- Implémenter une évolution sans RFC validé
- Supprimer un RFC (il est conservé pour traçabilité)

## Liens

- **00_CONSTITUTION** : un RFC peut modifier la Constitution
- **03_DECISIONS** : un RFC adopté produit une décision
- **01_DIRECTIVES** : une Directive peut initier des RFC
