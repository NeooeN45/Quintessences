## DEC-000012 — Encyclopédie de l'Écosystème

### Ce qui change

liste :

| Sujet | Technologie candidate | ADR |
|---|---|---|
| Base graphe | Neo4j | ADR-0008 |
| Triple store | Apache Jena | ADR-0009 |
| Recherche full-text | Elasticsearch | ADR-0010 |
| API | GraphQL + REST | ADR-0011 |
| Pipeline ingestion | Apache Airflow | ADR-0012 |
| NLP / extraction | spaCy + transformers + LLM | ADR-0013 |

**Amendement** : ces ADR sont remplacés par les ADR-001 à ADR-006 de la
RFC-0011 :

| ADR original | ADR v6.1 | Changement |
|---|---|---|
| ADR-0008 (Neo4j) | ADR-003 (AGE benchmark) | Neo4j différé, AGE évalué |
| ADR-0009 (Jena) | — (différé) | Projection régénérable, pas prioritaire |
| ADR-0010 (ES) | — (différé) | PG trigram + GIN en Vague 1 |
| ADR-0011 (GraphQL) | — (différé) | REST seul en Vague 1 |
| ADR-0012 (Airflow) | — (différé) | Pipelines en Python natif Vague 1 |
| ADR-0013 (NLP) | — (différé) | Extraction en Vague 2+ |

Nouveaux ADR : ADR-001 (Resource), ADR-002 (GSIE Temporal & Provenance Engine), ADR-004
(migration), ADR-005 (Outbox), ADR-006 (object storage).

### Ce qui est conservé

- L'Encyclopédie est le **produit principal** de GSIE : inchangé
- L'échelle visée **million d'entrées minimum** : inchangée
- Le périmètre **tout l'écosystème** : inchangé
- Les identifiants stables `GSIE-XXX` : inchangés
- Les garde-fous (S-1 à S-7, qualité > quantité) : inchangés

## DEC-000019 — Architecture Phase 4 + plan 24 semaines

### Ce qui change

DEC-000019 définit un plan en 6 vagues (24 semaines). Le métamodèle
v6.1 ajoute une **Vague 0** (gouvernance + RFC + ADR + audit migration)
avant la Vague 1, et étend la Vague 1 pour implémenter les 65 types
complets.

**Plan amendé** :

| Vague | Durée | Contenu | Changement |
|---|---|---|---|
| 0 | ~2 semaines | RFC + DEC + 6 ADR + tests contractuels + contrats interface + audit migration | **Nouvelle** |
| 1 | 4 semaines (étendues) | 65 types + migration schéma + benchmark AGE + Essence 360° | **Étendue** (65 types dès le départ, arbitrage T3) |
| 2 | 4 semaines | Moteurs domaine Python (GIS, Climate, Botanical, Pedology) | Inchangée |
| 3-6 | 16 semaines | Cœur raisonnement + avancés + validation + learning | Inchangées |

### Ce qui est conservé

- L'architecture multi-langage (Python + Rust + Go différé) : inchangée
- L'architecture API (FastAPI + PostGIS + Redis) : inchangée
- Le respect du graphe de dépendances (livrable 204) : inchangé
- Les garde-fous (CON-001, CON-005, profiler avant Rust) : inchangés
- Le plan de repli Python si Rust bloque : inchangé

## DEC-000020 — Knowledge Engine Semaine 3

### Ce qui change

DEC-000020 acte le Knowledge Engine en **Python pur avec stockage en
mémoire** pour la Vague 1, avec persistance PostgreSQL/Neo4j comme
"évolution future".

**Amendement** : la persistance n'est plus une "évolution future" mais
un objectif de la **Vague 0/1**. Le store in-memory (`self._store:
dict`) est remplacé par un repository PostgreSQL sur le schéma v6.1
(`resource` + `assertion` + tables associées). Neo4j est différé (AGE
benchmark).

### Ce qui est conservé

- Les 5 types de requête (par_concept, par_relation, par_domaine,
  par_essence, par_station) : préservés (adaptés à `claim_kind`)
- Le versionnement CON-010 : préservé via `Revision`
- L'auth JWT, rate limiting, observabilité : inchangés
- Les 33 tests existants : adaptés au nouveau schéma (Vague 0)
- Le pipeline Evidence → Knowledge (DEC-000021) : préservé, adaptateur
  Rust évalue + Python enrichit (arbitrage T4)

### Transition

| Élément | État actuel | État Vague 1 |
|---|---|---|
| Store | `dict[UUID, KnowledgeObject]` | Repository PG sur schéma v6.1 |
| Schéma | `knowledge_models.py` (non connecté) | `resource` + 42 tables (ADR-001) |
| Migration 0001 | Tables `knowledge_objects` + AGE inerte | Migration 0002 : transformation v6.1 (ADR-004) |
| Evidence | `evidence_level` direct sur KO | `EvidenceAssessment` (adaptateur Rust+Python) |
| 67 tests Rust | Intacts | Intacts (arbitrage T4) |
| 33 tests Python Knowledge | Adaptés | Adaptés au schéma v6.1 |

## DEC-000021 — Pipeline intégré Evidence → Knowledge

### Ce qui change

DEC-000021 acte le pipeline intégré Evidence→Knowledge en Vague 1. Le
pipeline transforme des `EvidenceResult` (Rust) en `KnowledgeObject`
(Python) avec `evidence_level` direct.

**Amendement** : le pipeline est adapté au schéma v6.1 :

| Élément DEC-000021 | Élément v6.1 |
|---|---|
| `EvidenceResult` → `KnowledgeObject` | `EvidenceResult` → `EvidenceAssessment` sur `Assertion` |
| `evidence_level` direct sur KO | `EvidenceAssessment.level` (A-F) |
| Pipeline produit un KO | Pipeline produit une `Assertion` + son `EvidenceAssessment` |
| Adaptateur Rust+Python | Préservé (arbitrage T4) : Rust évalue A-F, Python enrichit (evaluator, method, date, scope) |

### Ce qui est conservé

- Le pipeline intégré Evidence→Knowledge reste valide conceptuellement.
- Les 11 tests de pipeline sont adaptés au schéma v6.1.
- Les 67 tests Rust Evidence Engine sont intacts.
- L'ordre Evidence → Knowledge (chaîne principale) est préservé.
